2. Create `root/player_vault/` (empty directory).
3. Open `root/campaign.sqlite` via `open_campaign_db`, run `apply_migrations`, read `current_schema_version` → `schema_version_value`.
4. Insert row into `campaigns` (campaign_id = `generate_campaign_id(slug)`, slug = `slugify(name)`, etc.).
5. Write `ProviderSettings(...)` through `SettingsRepository.put_provider_settings(campaign_id, ...)`. All writes in ONE transaction (`with conn:` block).
6. Serialize `CampaignManifest` to TOML at `root/campaign.toml` using `tomli_w` if present in deps; **the agent's Discretion**: since `tomli_w` is NOT in pyproject.toml, write a minimal hand-rolled TOML writer inside `campaign.py` that supports only `str`, `int`, `bool` top-level fields (no nesting). <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI stdin → init/configure | User-supplied name/path/env-var-name is untrusted. Slug validation + path resolution via `Path` prevent traversal into parent dirs. |
| Environment variable → SecretRef → configure → SQLite | Only the env var NAME is stored; the VALUE is never read by configure (it's resolved on demand by the provider client in Plan 02-05). Ensures the secret never touches the DB. |
| Campaign directory layout → open_campaign | A malformed or partial campaign dir (missing file, corrupted TOML) must produce a specific typed error message to the CLI, not a traceback. |
| SettingsRepository.put → SQLite | All JSON payloads pass through RedactionCanary before write. Same invariant as turn_close.py. |
| campaign.toml on disk | Plain text, intended for user inspection. MUST NOT contain secret material, only `SecretRef` descriptors with env var names or keyring identifiers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Arbitrary path passed to `sagasmith play --campaign <path>` | mitigate | `open_campaign` validates that `campaign.toml`, `campaign.sqlite`, `player_vault/` all exist; otherwise exit 2 with specific message. User-trusted local path, not a remote URL. |
| T-03-02 | Tampering | User hand-edits campaign.toml with an invalid campaign_slug | mitigate | `open_campaign` calls `CampaignManifest.model_validate`, which runs the regex validator; ValueError propagates to CLI with exit 2. |
| T-03-03 | Repudiation | No audit trail for settings changes | accept | Phase 3 MVP scope; Phase 7 adds settings change history. Current `updated_at` column supports future audit. |
| T-03-04 | Information Disclosure | Configure command accidentally echoes resolved env-var value | mitigate | Command only accepts `api_key_ref` (the reference), never the value. Test `test_configure_never_echoes_secret` asserts the secret value does not appear in stdout/stderr. |
| T-03-05 | Information Disclosure | Secret written to SQLite settings table | mitigate | `SettingsRepository.put` runs `RedactionCanary` on JSON bytes before INSERT; raises `TrustServiceError` on any hit. Mirrors turn_close.py invariant. |
| T-03-06 | Information Disclosure | Secret written to campaign.toml | mitigate | Manifest schema has no `api_key` field; `CampaignManifest` covers only identity metadata. ProviderSettings (which has the SecretRef) lives in SQLite, not TOML. |
| T-03-07 | Denial of Service | `init` crashing on existing directory wipes data | mitigate | Use `mkdir(parents=True, exist_ok=False)` — raises FileExistsError before any write. CLI catches and prompts; no destructive fallback. |
| T-03-08 | Elevation of Privilege | `configure` allows changing provider to `fake` to bypass CostGovernor | accept | Per MVP scope: user owns their campaign; `fake` provider is intentional for demo/test. Phase 4+ graph runtime enforces preflight regardless of provider. |
| T-03-09 | Tampering | Concurrent `sagasmith init` races on same path | accept | Local-first single-user app; documented assumption is one CLI instance per campaign. `exist_ok=False` provides best-effort guard. |
