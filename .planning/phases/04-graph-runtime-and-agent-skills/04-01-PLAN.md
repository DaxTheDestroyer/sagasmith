<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SagaGraphState ↔ SagaState | TypedDict/Pydantic round-trip — every checkpoint/resume crosses this boundary. Invalid data must be caught at `to_saga_state()` before graph consumes it. |
| Agent node ↔ AgentServices | Nodes receive injected services; the boundary is enforced by thin-node rule (no direct SQLite, no direct httpx). |
| pending_player_input ↔ rules_lawyer_node | Raw player string enters a trigger-phrase parser. Even in this stub, no string value may propagate into a SQL/shell context. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | SagaGraphState field drift from SagaState | mitigate | Import-time assertion in `graph/state.py` raises RuntimeError on key-set mismatch; CI catches any schema evolution that forgets to update the TypedDict. |
| T-04-02 | Tampering | Routing exhaustiveness gap | mitigate | Import-time guard in `graph/routing.py` verifies `PHASE_TO_ENTRY` covers every `Phase` enum literal. New phase without a route fails at module load. |
| T-04-03 | Denial of Service | Unbounded loop in onboarding self-loop | mitigate | Onboarding node returns `phase="onboarding"` only when required profile fields are missing. Plan 04-02 will add a turn-count interrupt as a secondary brake. |
| T-04-04 | Information Disclosure | `pending_player_input` echoed to logs via node return | accept | Nodes in this plan do not log. Plan 04-02 introduces the agent/skill activation log and MUST apply RedactionCanary to any transcript content before persistence (documented constraint for the next plan). |
| T-04-05 | Elevation of Privilege | Node reaches into SQL or LLM directly | mitigate | Thin-node rule enforced by test 5 (purity) + code review: nodes only accept `services` and never import `sqlite3` or `httpx`. Pyright detects forbidden imports via ruff isort-local rules. |
| T-04-06 | Repudiation | Non-deterministic rules_lawyer_node results | mitigate | DiceService is injected; seeded-replay test proves same inputs → same outputs. Plan 04-02 captures this in the agent activation log. |

</threat_model>
