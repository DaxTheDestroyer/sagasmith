<objective>
Wire LangGraph's native interrupt/resume primitives to the TUI command layer and prove the full TUI→graph→narration path with an end-to-end smoke test. Delivers GRAPH-04 (interrupts represented as graph primitives, not ad-hoc UI branches) while keeping Phase 3 SafetyEvent regressions green.

Purpose: The reviews flagged HIGH risk on the prior approach (shadow `pending_interrupt` state field) and MEDIUM risk on budget-stop coupling to nodes. This plan uses LangGraph's native `interrupt()` and `Command(resume=...)` primitives (proven accessible by Plan 04-02's spike test) and keeps `BudgetStopError` handling in the runtime wrapper. It also ships the end-to-end smoke test the consensus review requested.
Output: Interrupt envelope model, TUI commands that post interrupts when a graph is bound, runtime-level budget translation, and a smoke test that runs one play turn from TUI input to stub narration to /pause interrupt.

## Revision notes (from 04-REVIEWS.md)

This plan was extracted from the original Plan 04-02 Task 2 after reviewer consensus on task-split needs. Key choices:
- **Native LangGraph primitives**: interrupts go through `Command(resume=<InterruptEnvelope dict>)` on the graph's thread; the runtime's `post_interrupt` calls `graph.update_state(config, {"last_interrupt": envelope.model_dump()})` (a state field, not a shadow short-circuit mechanism) and triggers a resume. Nodes check `state.get("last_interrupt")` on next run — this is the natural LangGraph pattern per Context7 docs.
- **RetconCommand acknowledge-only**: Phase 8 owns retcon confirmation + rollback. Posting an interrupt today would halt the turn without a resume path.
- **BudgetStopError translation at runtime**: `GraphRuntime.invoke_turn` wraps the graph.invoke in try/except; a BudgetStopError raised by any node's cost-aware code (Phase 6+) becomes an `InterruptKind.BUDGET_STOP` interrupt without teaching nodes about interrupts.
- **`/pause` + `/line` dual-write**: preserve Phase 3 SafetyEvent writes AND post graph interrupts. Graceful degrade when `app.graph_runtime is None`.
- **End-to-end smoke test**: `tests/integration/test_tui_graph_smoke.py` proves the full path works.
</objective>

LangGraph 1.0.x interrupt API (verified via Context7 spike Task 1):
```python
from langgraph.types import Command, interrupt

# Inside a node: pause and surface value to caller
response = interrupt({"kind": "pause", "reason": "/pause"})

# Caller inspects and resumes:
graph.invoke(Command(resume={"ack": True}), config)
```

Preferred approach for this plan: rather than adding `interrupt()` calls inside nodes (which reviewers flagged as premature complexity for Phase 4 stubs), use `graph.update_state(config, {"last_interrupt": envelope})` to record the interrupt in state, then `graph.invoke(Command(resume=None), config)` to resume the thread. The TUI uses `runtime.resume_after_interrupt()` to continue after the user clears the condition. This is still native LangGraph — `update_state` + `Command(resume=...)` are first-class APIs.

    ```python
    """Graph interrupt envelopes + canary guard.

    Interrupts use LangGraph's native update_state + Command(resume=...) pattern:
    - `post_interrupt` writes an envelope to state via `graph.update_state`.
    - `resume_after_interrupt` clears it and resumes the thread.

    This is NOT a shadow mechanism — it's the documented LangGraph API for
    passing caller-side signals through to graph execution. See 04-REVIEWS.md
    consensus change #1.
    """
    from dataclasses import dataclass, field, asdict
    from datetime import datetime, UTC
    from enum import StrEnum
    from typing import Any
    from sagasmith.services.errors import TrustServiceError


    class InterruptKind(StrEnum):
        PAUSE = "pause"
        LINE = "line"
        RETCON = "retcon"      # defined but not posted in Phase 4 — Phase 8 scope
        BUDGET_STOP = "budget_stop"
        SESSION_END = "session_end"


    @dataclass(frozen=True)
    class InterruptEnvelope:
        kind: InterruptKind
        payload: dict[str, Any]
        thread_id: str
        created_at: str

        def model_dump(self) -> dict:
            return {
                "kind": str(self.kind),
                "payload": self.payload,
                "thread_id": self.thread_id,
                "created_at": self.created_at,
            }

        @classmethod
        def build(cls, *, kind: InterruptKind, payload: dict | None, thread_id: str, canary=None) -> "InterruptEnvelope":
            payload = payload or {}
            # Redaction canary guard
            if canary is None:
                from sagasmith.evals.redaction import RedactionCanary
                canary = RedactionCanary()
            import json
            if canary.scan(json.dumps(payload)):
                raise TrustServiceError(f"interrupt payload for kind={kind} contains redacted content")
            return cls(
                kind=kind,
                payload=payload,
                thread_id=thread_id,
                created_at=datetime.now(UTC).isoformat(),
            )


    def extract_pending_interrupt(runtime) -> dict | None:
        """Inspect the runtime's current graph state for a pending interrupt envelope."""
        snapshot = runtime.graph.get_state(runtime.thread_config)
        values = getattr(snapshot, "values", {}) or {}
        return values.get("last_interrupt")
    ```
