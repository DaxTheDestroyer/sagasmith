<objective>
Implement the Agent Skills adapter per ADR-0001: filesystem discovery, compact catalogs, on-demand skill loading, and per-skill activation logging. Ship the first-slice SKILL.md catalog for all five MVP agents plus three cross-cutting skills. Deterministic skills (degree-of-success, seeded-roll-resolution, skill-check-resolution, turn-close-persistence) get working handlers wired into their nodes; prompted skills ship as SKILL.md bodies for consumption by Phase 6's LLM-driven nodes. Delivers SKILL-01 through SKILL-05.

Purpose: ADR-0001 commits to a first-party adapter (estimated 150-300 LOC) to keep Agent Skills portable across orchestration frameworks and to enable per-skill eval fixtures. This plan ships that adapter plus the first-slice catalogs so Phases 5-8 can implement real skill handlers without re-deciding the adapter contract.
Output: `src/sagasmith/skills_adapter/` package, `src/sagasmith/skills/` cross-cutting SKILL.md files, per-agent `skills/` directories under `agents/<name>/`, updates to nodes that call `load_skill` + `set_skill(...)` for the deterministic first-slice skills, plus fixture-driven tests for the adapter.
</objective>

<interfaces>
Agent Skills SKILL.md format per https://agentskills.io/specification and ADR-0001:
```
---
name: scene-brief-composition
description: Build a structured SceneBrief for the next playable scene.
allowed_agents: [oracle]
inputs: [SagaState, MemoryPacket, pending_conflicts]
outputs: [SceneBrief]
implementation_surface: prompted
success_signal: Every brief includes required fields and never contains player-facing narration.
---

# Scene Brief Composition

<Skill body in markdown — procedural instructions the agent reads AFTER it decides to activate this skill.>
```

YAML parser: We do NOT add a new YAML dependency. Pyright/ruff already allow manual parsing of frontmatter via `str.split('---')` and parsing the key-value block with a simple line parser (the frontmatter shape is small and fixed). Rationale: adding PyYAML for a handful of files violates the minimalism shown by Plan 03-01's hand-rolled TOML writer (STATE.md [03-01]).

From ADR-0001 §Directory layout:
```
agents/<name>/skills/<skill-name>/SKILL.md   # agent-scoped skills
skills/<skill-name>/SKILL.md                  # cross-cutting skills
```

From ADR-0001 §Skill adapter:
- `SkillStore` scans directories, parses frontmatter, exposes discovery metadata.
- `load_skill` tool returns full SKILL.md body as tool response.
- Catalog-injection helper renders skill list into each node's system prompt.
</interfaces>

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SKILL.md frontmatter ↔ SkillStore | Untrusted key/value pairs parsed from disk; must not allow injection into later prompts. |
| SKILL.md body ↔ agent LLM prompts (future) | Body content may eventually be embedded in LLM system prompts; must not carry secrets. |
| `allowed_agents` ↔ load_skill | An overly-permissive `allowed_agents: ["*"]` on a sensitive skill could allow cross-agent capability escalation. |
| Contextvar `_current_activation` ↔ concurrent turns | If the graph ever becomes multi-threaded, the contextvar must isolate turns correctly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-15 | Information Disclosure | SKILL.md body contains secret-shaped payload | mitigate | `SkillStore.scan` runs RedactionCanary over every body; canary hits become `SkillValidationError("redacted content")` and the skill is excluded. Error message does NOT echo the payload. |
| T-04-16 | Elevation of Privilege | Skill with `allowed_agents: ["*"]` granted broader access than intended | mitigate | Only cross-cutting skills under `src/sagasmith/skills/` may declare `["*"]`. Adapter enforces: if a skill is discovered under `agents/<agent>/skills/` and has `allowed_agents: ["*"]`, downgrade to `[agent_name]` and record a warning (future review candidate). |
| T-04-17 | Tampering | Hand-rolled frontmatter parser misinterprets edge-case YAML | mitigate | Tests cover: multi-line descriptions (single-line only, multi-line raises SkillValidationError), list tokens with commas, missing separator, extra keys. Any unrecognized structure fails closed with SkillValidationError. |
| T-04-18 | Information Disclosure | `set_skill` logs skill name that leaks content | mitigate | `set_skill` regex guard (from Plan 04-02 T-04-09) enforces kebab-case filesystem names only. Anything else → ValueError before logger writes. |
| T-04-19 | Denial of Service | Misbehaving SKILL.md with 10MB body exhausts memory | mitigate | SkillStore rejects any file > 256KB with SkillValidationError (test fixture included). This bounds catalog + load memory usage. |
| T-04-20 | Repudiation | Concurrent turns overwrite each other's contextvar | accept | Textual is single-threaded per Plan 03-04 decision; LangGraph in this MVP does not parallelize nodes within a single turn. If Phase 6+ introduces node-level parallelism, revisit (noted in 04-03 SUMMARY). |
