## Implementation Steps

### Step 1: Implement Safety Pre-Gate Service (D-06.3)

**File:** `src/sagasmith/services/safety_pre_gate.py`

Create the pre-generation safety service that returns a **routing verdict**, not raises an exception. Verdict type:

```python
class PreGateVerdict:
    Allowed: ClassVar  # proceed unchanged
    Rerouted(new_intent: str, reason: str)  # substitute and proceed
    Blocked(reason: str)  # halt and post SAFETY_BLOCK interrupt
```

Behavior:
- Compile keyword/regex patterns from `ContentPolicy.hard_limits` once at service init
- `check(intent: str, policy: ContentPolicy) -> PreGateVerdict`
- Hard-limit matches → `Blocked`
- Soft-limit matches with `fade_to_black` / `avoid_detail` → `Rerouted` with redacted intent
- Soft-limit matches with `ask_first` → `Blocked` with reason "needs_player_consent"
- Follows the `RedactionCanary` pattern-match model from `evals/redaction.py`
- Integrates with Oracle's `content-policy-routing` skill (Task 2 Step 10)
- Logs every non-`Allowed` verdict via `SafetyEventService`

The Oracle node — not the pre-gate itself — posts `InterruptKind.SAFETY_BLOCK` via `runtime.post_interrupt(...)` when it receives a `Blocked` verdict. The pre-gate stays a pure function so it can be unit-tested without a runtime.

### Step 2: Implement Safety Post-Gate Service (D-06.1)

**File:** `src/sagasmith/services/safety_post_gate.py`

The post-generation safety service that:
- Uses cheap LLM classifier (`cheap_model` from `ProviderSettings`) for content violations
- `scan(prose: str, policy: ContentPolicy) -> PostGateVerdict` returns `Pass` | `Rewrite(reason)` | `BlockFallback(reason)`
- Two-rewrite limit is enforced **by the Orator pipeline** (Task 4), not by this service — keeps the service stateless
- Integrates with Orator scene rendering as the gate after the inline matcher passes
- Logs all safety events via `SafetyEventService`
- Honors per-turn budget per D-06.6 — each scan counts against the same budget as rendering
