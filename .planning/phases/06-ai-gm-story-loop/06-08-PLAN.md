## Testing Scenarios

### Scenario 1: Standard Narrative Flow
- Player starts new campaign
- World bible and campaign seed generated
- First scene brief created
- Player takes narrative action
- RulesLawyer processes intent
- Orator generates narration
- Turn completes successfully

### Scenario 2: Mechanical Action Flow
- Player initiates skill check
- Intent parsed and proposal generated
- Deterministic resolution executed
- Results narrated appropriately
- Dice UX mode respected

### Scenario 3: Combat Flow
- Player starts combat
- Initiative determined
- Combat actions resolved
- State tracked correctly
- Combat conclusion narrated

### Scenario 4: Safety Intervention - Pre-Gate
- Player input triggers hard limit
- Oracle blocks scene generation
- Appropriate safety event logged
- Player notified of block

### Scenario 5: Safety Intervention - Post-Gate
- Orator generates prohibited content
- Post-gate detects violation
- Rewrite attempted
- Fallback activated if needed
- Safety event logged

### Scenario 6: Narration Recovery
- Narration generation interrupted
- Pre-narration checkpoint restored
- Deterministic outcomes preserved
- New narration generated
- Turn completes successfully

### Scenario 7: Turn Discard
- Incomplete turn detected
- Player chooses to discard
- State reverted to pre-narration
- No side effects from discarded turn
