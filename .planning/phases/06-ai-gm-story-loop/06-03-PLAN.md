# Task 3: Implement Intent-to-Proposal Layer (RulesLawyer)

**Task ID:** 06-03
**Owner:** Kilo (AI Assistant)
**Phase:** 06-ai-gm-story-loop
**Requirements:** AI-06, D-16, D-17 (AI-12 is already satisfied in Phase 4 — agent skill activation logging exists; this task only consumes that infrastructure, it does not deliver against AI-12)

## Overview

Add an LLM layer to the RulesLawyer that converts natural language player input into mechanical proposals before passing to deterministic services. This maintains the trust boundary where LLMs propose but deterministic services validate and execute.

## Implementation Steps

### Step 1: Implement Intent Resolution Service

**File:** `src/sagasmith/services/intent_resolution.py`

Create the intent resolution service that:
- Parses natural language player input
- Identifies potential mechanical actions
- Uses hybrid approach: **deterministic regex/pattern match first**, LLM fallback only when no pattern matches
- Returns ranked intent candidates with confidence scores
- Honors per-turn budget per D-06.6: on `BudgetStopError`, falls through to deterministic-only routing (the player sees a hint to use explicit `/check athletics 15` syntax) rather than failing the turn

### Step 2: Enhance CheckProposal Model

**File:** `src/sagasmith/schemas/mechanics.py`

Ensure CheckProposal model includes all required fields:
- id: str
- reason: str
- kind: Literal["skill", "attack", "save", "initiative", "flat"]
- actor_id: str
- target_id: str | None
- stat: str
- modifier: int
- dc: int | None
- secret: bool

### Step 3: Implement Intent-to-Proposal Logic

**File:** `src/sagasmith/agents/rules_lawyer/intent_to_proposal.py`

Create the implementation that:
- Takes player input and scene context
- Calls intent resolution service
- Converts identified intents to CheckProposal objects
- Uses LLM structured calls when deterministic parsing fails
- Validates proposals against CheckProposal schema

### Step 4: Update RulesLawyer Node

**File:** `src/sagasmith/agents/rules_lawyer/node.py`

Modify the rules_lawyer_node function to:
- Add LLM intent parsing layer before deterministic resolution
- Call intent-to-proposal logic for natural language input
- Maintain existing deterministic resolution for actual mechanics
- Add routing to skip mechanics when no action is detected
- Add proper skill activation logging

### Step 5: Implement Skill Check Resolution Skill

**File:** `src/sagasmith/agents/rules_lawyer/skills/skill-check-resolution/SKILL.md`

Create or update the skill definition with:
- Name: skill-check-resolution
- Description: Convert intent to skill check proposal and resolve with deterministic services
- Inputs: CheckProposal, CharacterSheet, scene context
- Output: CheckResult
- Implementation surface: hybrid
- Success signal: Valid CheckResult with auditable roll

### Step 6: Implement Combat Resolution Skill

**File:** `src/sagasmith/agents/rules_lawyer/skills/combat-resolution/SKILL.md`

Create or update the skill definition with:
- Name: combat-resolution
- Description: Convert combat intent to attack proposals and resolve with deterministic services
- Inputs: CheckProposal, CombatState, CharacterSheet
- Output: CheckResult, updated CombatState
- Implementation surface: hybrid
- Success signal: Valid combat resolution with state updates
