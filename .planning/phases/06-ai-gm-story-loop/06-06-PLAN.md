# Task 6: Narration Discard + Recovery Commands (Graph + TUI)

**Task ID:** 06-06
**Owner:** Kilo (AI Assistant)
**Phase:** 06-ai-gm-story-loop
**Requirements:** GRAPH-06, GRAPH-07, D-13, D-14

## Overview

Add the missing pieces for narration recovery. **Pre-narration checkpointing already exists** in [graph/runtime.py:159-170](src/sagasmith/graph/runtime.py#L159-L170), and incomplete-narration detection already exists in [graph/runtime.py:69-77](src/sagasmith/graph/runtime.py#L69-L77) via `snapshot.next == ("orator",) and snapshot_turn_id == turn_id`.

The actual gaps are:
1. A discard flow that rewinds to the pre-narration checkpoint and marks the turn as discarded.
2. A retry flow that rewinds and re-invokes the Orator.
3. `TurnRecord` status transitions for both flows.
4. TUI commands so the player can trigger them.
5. A regression test pinning byte-identical deterministic state across happy / retry / discard paths.

The plan deliberately does NOT re-spec checkpoint restoration or pre-narration write logic — those are done.

## Implementation Steps

### Step 1: Add `discard_incomplete_turn` to GraphRuntime

**File:** `src/sagasmith/graph/runtime.py`

Add a method `discard_incomplete_turn(turn_id: str) -> TurnRecord` that:
- Loads `CheckpointRef(kind="pre_narration")` for `turn_id` via `CheckpointRefRepository`
- Calls `self.graph.update_state(...)` (or equivalent LangGraph rewind primitive) to point the thread at that checkpoint
- Updates the `TurnRecord.status` to `discarded` and `completed_at` to now
- Commits and returns the updated record

Raise a clear error if no pre-narration checkpoint exists for the turn.

### Step 2: Add `retry_narration` to GraphRuntime

**File:** `src/sagasmith/graph/runtime.py`

Add `retry_narration(turn_id: str) -> dict[str, Any]` that:
- Loads the same `pre_narration` `CheckpointRef`
- Rewinds the thread state to that checkpoint
- Clears `pending_narration` in the rewound state (defensive — the pre-narration snapshot should already have it empty)
- Re-invokes the graph past the `orator` interrupt and through `archivist`
- Returns the final state values

The deterministic invariant: `check_results`, `scene_brief`, dice roll outcomes, and HP/state must be byte-identical to a non-retried run.

### Step 3: TurnRecord status transitions

**File:** `src/sagasmith/schemas/persistence.py` (if status is a Literal) and `src/sagasmith/persistence/repositories.py`

Add status values: `narrated`, `discarded`, `retried`. The existing values (`needs_vault_repair`, etc.) remain. Document the state machine:

```
in_progress  --(orator complete)--> narrated
in_progress  --(user /retry)----->  retried   (then back to in_progress for next attempt)
in_progress  --(user /discard)--->  discarded
```

### Step 4: Regression test for deterministic stability

**File:** `tests/integration/test_narration_recovery.py`

Single test using `DeterministicFakeClient`:
1. Run a turn end-to-end; capture `check_results`, `scene_brief`, `world_bible` snapshot, dice ledger.
2. Run the same turn but call `retry_narration` once before letting it complete.
3. Run the same turn but call `discard_incomplete_turn`, then run a fresh turn with the same input.

Assertions:
- `check_results` and dice ledger are byte-identical between (1) and (2).
- After (3), the discarded turn leaves no orphan transcript or roll rows.

### Step 5: TUI recovery commands

**File:** `src/sagasmith/tui/commands/recovery.py`

Two commands:
- `/retry` — calls `runtime.retry_narration(current_turn_id)`. Disabled unless TUI state shows an incomplete narration.
- `/discard` — calls `runtime.discard_incomplete_turn(current_turn_id)`. Confirms first.

Register both in `tui/commands/registry.py`.
