# Task 2: Implement Scene Brief Composition (Oracle)

**Task ID:** 06-02
**Owner:** Kilo (AI Assistant)
**Phase:** 06-ai-gm-story-loop
**Requirements:** AI-01, AI-02, AI-04, D-07, D-08, D-09

## Overview

Replace the stub SceneBrief with a real LLM-powered implementation that generates scene plans based on player input, memory context, and campaign state. This enables dynamic story generation that adapts to player choices.

## Implementation Steps

### Step 1: Beat ID Representation (D-06.4)

**File:** `src/sagasmith/schemas/narrative.py`

The `SceneBrief` model already has every field listed in `STATE_SCHEMA.md` — `scene_id`, `intent`, `location`, `present_entities`, `beats`, `success_outs`, `failure_outs`, `pacing_target`, `callbacks_seeded`, `callbacks_payoff_candidates`, `mechanical_triggers`, `content_warnings`. **No additions needed for the existing fields.**

What this step DOES:
- Decide between extending `beats: list[str]` to be IDs (e.g. `"beat_001"`, `"beat_002"`) or adding a parallel `beat_ids: list[str]` and keeping `beats` as human-readable text. Recommendation: **add `beat_ids`** so the LLM-generated `beats` strings stay readable for prompt audits.
- Add the corresponding field to `SagaState` / `SagaGraphState`: `resolved_beat_ids: list[str] = Field(default_factory=list)` populated by the Orator (per Task 4 D-06.4).
- Update the JSON schema fixtures and snapshot tests under `tests/schemas/` and `tests/fixtures/`.

### Step 2: Implement Scene Brief Composition Skill

**File:** `src/sagasmith/agents/oracle/skills/scene-brief-composition/SKILL.md`

Update the existing skill definition with:
- Detailed procedure for LLM-based scene brief generation
- Input requirements: MemoryPacket, ContentPolicy, PlayerProfile
- Output requirements: Valid SceneBrief object
- Success signal: Every brief includes required fields and never contains player-facing narration

### Step 3: Implement Player Choice Branching Skill

**File:** `src/sagasmith/agents/oracle/skills/player-choice-branching/SKILL.md`

Update the existing skill definition with:
- Procedure for detecting player bypasses/rejections
- Input requirements: player_input, prior_scene_brief, memory_packet
- Output requirements: revised scene intent or next brief request
- Success signal: Bypass fixtures produce coherent replans without forcing original beat

### Step 4: Implement Content Policy Routing Skill

**File:** `src/sagasmith/agents/oracle/skills/content-policy-routing/SKILL.md`

Update the existing skill definition with:
- Procedure for pre-generation content filtering
- Input requirements: scene_intent, ContentPolicy
- Output requirements: allowed scene intent or reroute
- Success signal: Redlined fixture intents are rerouted before Orator sees them

### Step 5: Implement Scene Brief Composition Logic

**File:** `src/sagasmith/agents/oracle/skills/scene-brief-composition/logic.py`

Create the implementation that:
- Takes state context (MemoryPacket, ContentPolicy, PlayerProfile) as input
- Constructs appropriate LLM prompt with context
- Uses invoke_with_retry for structured JSON call
- Validates response against SceneBrief schema
- Handles errors and retries appropriately

### Step 6: Implement Player Choice Branching Logic

**File:** `src/sagasmith/agents/oracle/skills/player-choice-branching/logic.py`

Create the implementation that:
- Analyzes player input against current scene brief
- Detects bypasses, rejections, or reframing
- Generates appropriate replan request
- Uses lightweight processing where possible

### Step 7: Implement Content Policy Routing Logic

**File:** `src/sagasmith/agents/oracle/skills/content-policy-routing/logic.py`

Create the implementation that:
- Uses keyword/pattern matching against ContentPolicy
- Blocks or reroutes scene intents that violate hard limits
- Adjusts content for soft limits
- Follows the RedactionCanary pattern-match model

### Step 8: Update Oracle Node for Scene Management

**File:** `src/sagasmith/agents/oracle/node.py`

Modify the oracle_node function to:
- Trigger scene-brief composition when **any** of: `scene_brief is None`, `set(state["resolved_beat_ids"]) >= set(scene_brief.beat_ids)` (all beats resolved), or `player-choice-branching` flagged a bypass on the prior turn
- Otherwise pass through unchanged (scene continues)
- Honor per-turn budget per D-06.6: on `BudgetStopError`, return prior `scene_brief` unchanged and post `BUDGET_STOP` interrupt
- Add proper skill activation logging

### Step 9: Implement Conditional Routing Logic

**File:** `src/sagasmith/graph/graph.py`

Add conditional edges for Oracle routing:
- Route to oracle when `scene_brief is None`, all `beat_ids` are in `resolved_beat_ids`, or bypass detected (D-06.4)
- Skip Oracle when existing `scene_brief` is still valid

**File:** `src/sagasmith/graph/routing.py`

Update routing logic to handle new Oracle conditions. Routing predicates should be pure functions of state (no side effects, no LLM calls) so they can be unit-tested without mocks.

### Step 10: Pre-gate Integration (D-06.3)

**File:** `src/sagasmith/agents/oracle/node.py`

Before invoking `scene-brief-composition`, call `safety_pre_gate.check(intent, content_policy)` (Task 7). On the result:
- `Allowed` → proceed
- `Rerouted(new_intent)` → use `new_intent` and log a `SafetyEvent`
- `Blocked(reason)` → call `services.runtime.post_interrupt(InterruptKind.SAFETY_BLOCK, payload={"reason": reason})` and return state unchanged
