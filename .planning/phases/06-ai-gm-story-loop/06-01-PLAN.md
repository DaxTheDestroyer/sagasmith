# Task 1: Implement World Bible and Campaign Seed Generation (Oracle)

**Task ID:** 06-01
**Owner:** Kilo (AI Assistant)
**Phase:** 06-ai-gm-story-loop
**Requirements:** AI-01, AI-03, D-10, D-11, D-12

## Overview

Implement the world-bible-generation and campaign-seed-generation skills for the Oracle agent to create the initial world context and starting hooks. This is the player's first meaningful AI interaction in the campaign.

## Implementation Steps

### Step 1: Create WorldBible Pydantic Model

**File:** `src/sagasmith/schemas/world.py`

Create a Pydantic model for the WorldBible that includes:
- Core setting information (theme, tone, genre elements)
- Key locations with descriptions
- Important factions/NPCs
- Core conflicts and themes
- Magic/system rules context

### Step 2: Create CampaignSeed Pydantic Model

**File:** `src/sagasmith/schemas/campaign_seed.py` (new file — `campaign.py` already holds `CampaignManifest`/`ProviderSettings` and is unrelated)

Create a Pydantic model for the CampaignSeed that includes:
- List of 3-5 plot hooks aligned with player preferences
- Initial campaign arc structure
- Key characters and their motivations
- Initial conflicts and tensions

### Step 3: Implement World Bible Generation Skill

**File:** `src/sagasmith/agents/oracle/skills/world-bible-generation/SKILL.md`

Create the skill definition with:
- Name: world-bible-generation
- Description: Create the initial hidden campaign world bible from onboarding records
- Inputs: PlayerProfile, ContentPolicy, HouseRules
- Output: WorldBible object
- Implementation surface: prompted
- Success signal: Generated world bible validates and avoids hard limits

### Step 4: Implement Campaign Seed Generation Skill

**File:** `src/sagasmith/agents/oracle/skills/campaign-seed-generation/SKILL.md`

Create the skill definition with:
- Name: campaign-seed-generation
- Description: Produce 3-5 opening plot hooks and one selected seed arc
- Inputs: WorldBible, PlayerProfile
- Output: CampaignSeed object
- Implementation surface: prompted
- Success signal: Hook fixtures produce distinct hooks aligned with player preferences

### Step 5: Implement World Bible Generation Logic

**File:** `src/sagasmith/agents/oracle/skills/world-bible-generation/logic.py`

Create the implementation that:
- Takes PlayerProfile, ContentPolicy, HouseRules as input
- Constructs appropriate LLM prompt with context
- Uses invoke_with_retry for structured JSON call
- Validates response against WorldBible schema
- Handles errors and retries appropriately

### Step 6: Implement Campaign Seed Generation Logic

**File:** `src/sagasmith/agents/oracle/skills/campaign-seed-generation/logic.py`

Create the implementation that:
- Takes WorldBible and PlayerProfile as input
- Constructs appropriate LLM prompt with context
- Uses invoke_with_retry for structured JSON call
- Validates response against CampaignSeed schema
- Handles errors and retries appropriately

### Step 7: Update SagaState with New Fields

**File:** `src/sagasmith/schemas/saga_state.py`

Add new optional fields:
- world_bible: WorldBible | None
- campaign_seed: CampaignSeed | None

**File:** `src/sagasmith/graph/state.py`

Update SagaGraphState TypedDict to include:
- world_bible: object | None
- campaign_seed: object | None

### Step 8: Update Oracle Node Implementation

**File:** `src/sagasmith/agents/oracle/node.py`

Modify the oracle_node function to:
- Check if world_bible and campaign_seed are None
- If so, call world-bible-generation and campaign-seed-generation skills
- Store results in state
- Add appropriate skill activation logging

### Step 9: Add Conditional Routing Logic

**File:** `src/sagasmith/agents/oracle/node.py`

Add logic to:
- Skip world/campaign generation after first run (idempotent: re-entry must not regenerate)
- Only generate when in appropriate phase
- Handle errors gracefully

### Step 10: Trigger Wiring (worldgen handoff)

Worldgen is **not** triggered by a separate CLI command. Trigger:

- During Onboarding completion, `agents/onboarding/node.py` writes `PlayerProfile` and transitions phase to `play_ready`. The first invocation of `oracle_node` after that transition observes `world_bible is None` and `campaign_seed is None` and runs both skills.
- The graph routing in `graph/routing.py` already routes to `oracle` at the start of a turn when phase is `play_ready` (or equivalent). No new edges needed.
- Acceptance: starting a brand-new campaign and immediately calling `runtime.invoke_turn(...)` runs worldgen exactly once before the first `SceneBrief` composition.

### Step 11: Cost Governance (D-06.6)

Worldgen is one-shot per campaign and the most expensive single call in the phase. Per-campaign budget ceiling:
- Defined in `app/config.py` as `worldgen_max_usd` (default: $0.50; cheap-model fallback already in `invoke_with_retry`)
- Enforced via existing `BudgetStopError` path; on exhaustion, surface a setup-time error to the CLI/TUI rather than silently proceeding with partial data
