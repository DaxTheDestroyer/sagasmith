<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Deterministic service → Filesystem (vault) | Archivist writes canonical campaign knowledge to disk; any corruption or partial write violates source-of-truth guarantee. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | atomic_write tempfile leakage | mitigate | Use `os.replace()` which is atomic on POSIX and Windows (since Python 3.3); delete tempfile on error; validate YAML after replace |
| T-07-02 | Information Disclosure | GM-only fields accidentally written to player vault | accept (not in scope yet) | Player-vault-sync (Plan 07-05) will strip GM fields; this plan only writes master vault |
| T-07-03 | Repudiation | Vault pages written without audit trail | accept | Vault writes are not logged separately; SQLite turn record and checkpoint provide provenance; future audit may add file hashes |
| T-07-04 | Elevation of Privilege | Path traversal via crafted campaign_id | mitigate | Campaign_id derived from slug via `slugify`; only alphanumeric and dashes allowed; vault_path constructed with safe parts; no user-supplied raw path |
</threat_model>
