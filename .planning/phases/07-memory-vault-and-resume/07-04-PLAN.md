<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Archivist → LLM (rolling-summary-update) | LLM-generated summaries could inject speculation; treated as canonical after approval. |
| Session page generation → Player vault | Session pages are player-visible; incorrect metadata could misrepresent campaign history. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-12 | Tampering | Rolling summary LLM introduces non-canonical fact | mitigate | Summary update runs only at scene boundaries; uses constrained prompt to limit creative flourishes; summaries are validated by token cap and stored as canonical – future QA may add factuality spot-checks |
| T-07-13 | Information Disclosure | Session page accidentally includes GM notes if vault-page read with gm_only before sync | accept | Session page constructed only from player-known entities and public transcript; GM-only pages never imported into session page body. |
| T-07-14 | Denial of Service | Session page authoring blocks on DB query for large transcript | accept | MVP limits sessions to reasonable size; future paging may be added. |
</threat_model>
