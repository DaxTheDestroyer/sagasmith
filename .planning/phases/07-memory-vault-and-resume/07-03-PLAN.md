<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vault files → derived index (FTS5/NetworkX) | Derived layers are rebuildable; tampering with index should not affect canon. Accepted risk; indices can be regenerated. |
| Memory retrieval → LLM context | Retrieved content flows into LLM prompt; ensuring no GM-only content leaks here is critical (deferred to VAULT-03 sync). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-09 | Tampering | FTS5 index contains outdated entries after vault write failure | accept | Index is rebuilt from vault on demand by `ttrpg vault rebuild`; no trust assumption. |
| T-07-10 | Information Disclosure | MemoryPacket accidentally includes GM-only entity body text | mitigate | Entity resolution yields only entity IDs; full page body is NOT included in packet; only entity name and vault path. FTS5 snippets are limited to public pages (player_known or foreshadowed). GM-only pages excluded from index. (Enforced in FTS5 index_page filter). |
| T-07-11 | Denial of Service | Extremely large vault slows graph load | accept | Lazy loading with caching; user can run rebuild to optimize. |
</threat_model>
