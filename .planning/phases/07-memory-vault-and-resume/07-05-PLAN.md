<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI command → Vault service | User-triggered repair commands have elevated filesystem access; must not leak secrets or write outside campaign. |
| Vault sync → Player-visible files | Stripping must be thorough; any GM-only leakage violates spoiler safety. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-15 | Tampering | `ttrpg vault rebuild` runs with insufficient permissions and leaves partial indices | mitigate | Rebuild writes to temp files and uses atomic replace; on error, original index remains intact |
| T-07-16 | Information Disclosure | GM-only content included in session page because transcript contains GM notes | accept | Session page filters GM-only inline blocks via same regex as sync; only player-known entities included |
| T-07-17 | Repudiation | Player runs `vault sync` and erases GM-only content prematurely | accept | Player is the authority over their campaign; GM-only content is meant to stay hidden — sync does not alter master vault |
| T-07-18 | Elevation of Privilege | `vault rebuild` path traverse via malformed campaign_id | mitigate | Campaign IDs are slugified; paths constructed safely |

</threat_model>
