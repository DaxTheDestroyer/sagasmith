BudgetStopResult contract (to be created here):
```python
@dataclass(frozen=True)
class BudgetStopResult:
    blocked: bool
    cost_state: CostState  # projected state AFTER the would-be call
    worst_case_cost_usd: float
    message: str  # safe user-facing text, no secrets, no raw model names

    def raise_if_blocked(self) -> None:
        if self.blocked:
            raise BudgetStopError(self.message)
```

    Constructor: `CostGovernor(session_budget_usd: float, *, pricing_table: dict[str, PriceEntry] | None = None)`.
    - Validates `session_budget_usd >= 0`.
    - `pricing_table` defaults to `load_pricing_table()`.

    Property: `state -> CostState` (returns a deep copy / validated model to prevent external mutation).

    `record_usage(*, provider: str, model: str, usage: TokenUsage) -> CostUpdate` where `CostUpdate` is:
    ```python
    @dataclass(frozen=True)
    class CostUpdate:
        cost_usd: float
        cost_is_approximate: bool
        new_state: CostState
        warnings_fired_this_call: list[Literal["70", "90"]]
    ```
    - If `usage.provider_cost_usd is not None`: use it; `is_approximate=False`.
    - Else: call `estimate_cost_from_usage(provider, model, usage.prompt_tokens, usage.completion_tokens, self._table)`. On KeyError, set `cost_usd = 0.0, is_approximate = True` AND do not raise — cost unknown is better than crashing gameplay. (This is the agent's Discretion per CONTEXT.md; document the choice in an inline comment referencing D-09.)
    - Update internal state: `spent_usd_estimate += cost_usd`, `tokens_prompt += usage.prompt_tokens`, `tokens_completion += usage.completion_tokens`.
    - Threshold detection: compute `fraction = spent_usd_estimate / session_budget_usd` (guard divide-by-zero: if budget is 0, never fire warnings and never hard-stop beyond the initial state — a zero budget means "unlimited for dev tests" per pragma).
    - If `fraction >= 0.70` and `"70" not in warnings_sent`: append `"70"`, add to `warnings_fired_this_call`.
    - If `fraction >= 0.90` and `"90" not in warnings_sent`: append `"90"`, add to `warnings_fired_this_call`.
    - Warnings never fire twice (D-11).
    - Returns `CostUpdate`.

    `preflight(*, provider: str, model: str, prompt_tokens: int, max_tokens_fallback: int) -> BudgetStopResult`:
    - Compute worst-case `completion_tokens = max_tokens_fallback`.
    - Estimate `worst_case_cost_usd` via pricing table (static only; preflight never has provider-reported cost). If pricing key missing, treat `worst_case_cost_usd = 0.0` (unknown cost — cannot block on unknown; add inline comment referencing D-10 and note that a later plan may tighten this).
    - Projected `spent_after = spent_usd_estimate + worst_case_cost_usd`.
    - If `session_budget_usd > 0` AND `spent_after > session_budget_usd`: `blocked=True`, `message = f"Session budget ${session_budget_usd:.2f} would be exceeded by the next call (estimated +${worst_case_cost_usd:.4f}). Call not made."`, set `new_state.hard_stopped=True`.
    - Else `blocked=False`, `message = ""`, `new_state.hard_stopped=False`.
    - Returns `BudgetStopResult(blocked, projected_state, worst_case_cost_usd, message)`. This does NOT mutate `CostGovernor` state — the caller decides whether to apply.
    - If `blocked` is True: CALLER is expected to call `.raise_if_blocked()` or handle the result. Per D-12 the block must not cause a provider-side side effect — this method is pure (no network, no state mutation beyond returning the projected state).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TokenUsage → CostGovernor | Provider-reported usage is untrusted (could be spoofed by a bad provider); CostGovernor still operates on the claimed numbers but flags `is_approximate=False` only for provider-reported. |
| CostGovernor.preflight → caller → LLMClient | The caller MUST check `BudgetStopResult.blocked` (or call `.raise_if_blocked()`) before invoking the client. A later phase can add a wrapping helper that enforces this at the type level. |
| Internal CostState → external code | `state` property returns a validated copy; external mutation cannot affect governor. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-17 | Tampering | Warning-state manipulation | mitigate | `warnings_sent` append-only within governor; `state` property returns a fresh `CostState` — external mutation cannot affect internal state (tested). |
| T-02-18 | Repudiation | Unknown-model cost silently zero | accept | Documented tradeoff: unknown cost → 0.0 approximate so dev can keep playing; a later phase can add a stricter mode after the pricing table is comprehensive. |
| T-02-19 | Denial of Service | `CostGovernor` blocking valid play | mitigate | Preflight only blocks when `worst_case_cost_usd` is known AND `session_budget_usd > 0`; unknown cost never blocks; zero budget never blocks. |
| T-02-20 | Information Disclosure | `BudgetStopResult.message` content | mitigate | Message only formats `session_budget_usd` and `worst_case_cost_usd` — no model name, no agent, no prompt. Test asserts message excludes `"sk-"`, `"Bearer"`. |
| T-02-21 | Elevation of Privilege | Callers bypassing preflight | accept | Phase 2 scope is the service; the graph runtime in Phase 4 will enforce preflight-before-call at the graph edge. |
</threat_model>
