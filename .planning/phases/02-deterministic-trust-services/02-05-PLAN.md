OpenRouter API contract (external, stable — see https://openrouter.ai/docs):
- Base URL: `https://openrouter.ai/api/v1`
- `POST /chat/completions` — OpenAI-compatible chat completion.
- Request body keys used: `model`, `messages`, `temperature`, `max_tokens` (optional), `stream`, `response_format` (with `type: "json_schema"` and nested `json_schema: {name, schema, strict}` for structured).
- Response body keys used: `choices[0].message.content`, `choices[0].finish_reason`, `usage.prompt_tokens`, `usage.completion_tokens`, `usage.total_tokens`, `id` (provider_response_id), optionally `usage.cost` (OpenRouter-specific, when provided).
- Stream: Server-Sent Events. Each `data: ` line is a JSON chunk with `choices[0].delta.content` OR a `[DONE]` marker. A final usage object may arrive in the last data chunk.
- Auth header: `Authorization: Bearer <api_key>`.
- Recommended headers: `HTTP-Referer: https://github.com/sagasmith/sagasmith` (or leave unset), `X-Title: SagaSmith`.

HttpTransport Protocol (to be created):
```python
from collections.abc import Iterator
from typing import Protocol

@dataclass(frozen=True)
class HttpResponse:
    status_code: int
    text: str
    headers: dict[str, str]

class HttpTransport(Protocol):
    def post_json(self, *, url: str, headers: dict[str, str], json_body: dict[str, Any], timeout_seconds: int) -> HttpResponse: ...
    def post_stream(self, *, url: str, headers: dict[str, str], json_body: dict[str, Any], timeout_seconds: int) -> Iterator[str]: ...
```
`post_stream` yields each raw SSE `data:` line (with the `data: ` prefix stripped) including terminator lines; OpenRouterClient parses them.

HttpxTransport implementation: uses `httpx.Client()` with `http2=False`; `post_stream` uses `with client.stream("POST", ...) as resp: yield from resp.iter_lines()`, filtering to `data:`-prefixed lines.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OpenRouter API → OpenRouterClient | Response body is untrusted: could echo payload including headers (historical SaaS vulns). |
| OpenRouterClient → logger | Any error path must scrub API key and response body before persisting. |
| `api_key_ref` → resolved key → HTTP header | Key lifetime is the smallest possible: resolved inside method, dropped when method returns. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-22 | Information Disclosure | API key in error messages | mitigate | `_OpenRouterError` never carries body text; test asserts synthetic leak marker is absent from raised exception string. |
| T-02-23 | Information Disclosure | API key in stream failure event | mitigate | `FailedEvent.message` passes through `RedactionCanary`; any hit → substitute generic text; test proves substitution. |
| T-02-24 | Information Disclosure | API key in client logger | mitigate | Logger is passed the `ProviderLogRecord` from Plan 02-03 — which never contains request/response bodies. Test with synthetic key fixture asserts zero hits. |
| T-02-25 | Spoofing | Fake HttpxTransport used in prod | mitigate | `OpenRouterClient` accepts any `HttpTransport`; production bootstrap wires `HttpxTransport` explicitly. Phase 3 CLI will construct this; the fake is only in tests/conftest. |
| T-02-26 | Tampering | Malformed structured response | mitigate | JSON parse failure maps to `failure_kind="schema_validation"` so the Plan 02-03 retry ladder handles repair deterministically. |
| T-02-27 | Elevation of Privilege | Live-call test run accidentally in CI | mitigate | Opt-in only via `SAGASMITH_RUN_LIVE_OPENROUTER=1`; default CI never runs it per D-01. Add CI comment reminding maintainers never to set the env var in shared pipelines. |
</threat_model>
