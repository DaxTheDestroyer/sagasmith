LLMClient Protocol (src/sagasmith/providers/client.py):
```python
from typing import Protocol
from collections.abc import Iterator

class LLMClient(Protocol):
    def complete(self, request: LLMRequest) -> LLMResponse: ...
    def stream(self, request: LLMRequest) -> Iterator[LLMStreamEvent]: ...
```

Retry ladder (D-03) — implemented by free function `invoke_with_retry(client, request, *, cheap_model, agent_name, turn_id, logger) -> LLMResponse`:
- Attempt 1: call `client.complete(request)`. If returned `parsed_json` fails schema validation (when `response_format=="json_schema"`), record `retry_count=0` log, then:
- Attempt 2 (same-model repair): call `client.complete(request_with_repair_hint)` with `retry_count=1`.
- Attempt 3 (cheap-model repair): call `client.complete(request.model_copy(update={"model": cheap_model}))` with `retry_count=2`.
- After attempt 3 fails: emit final log with `failure_kind="schema_validation"`, `retry_count=3`, raise `ProviderCallError("schema validation exhausted retry ladder")` (from `sagasmith.services.errors`). The raised exception must NOT contain any prompt/response text.
- Network timeout and rate_limit failures follow LLM_PROVIDER_SPEC §7: single retry with same model (rate_limit uses provider-specified delay if supplied, capped at 30s), then `ProviderCallError`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLMClient.complete/stream → SagaSmith runtime | Everything the provider returns is untrusted until schema-validated. |
| Response text → provider log → SQLite/logs | Any prose that crosses into persisted logs must be scanned by RedactionCanary. |
| ProviderConfig.api_key_ref → provider client | Secret never flows through request or response models; only resolved in transport layer (Plan 05). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Spoofing | Fake client in production path | mitigate | `ProviderConfig.provider="fake"` is explicit; smoke/tests are the only code path that constructs `DeterministicFakeClient`. Plan 05 adds a runtime check that rejects `fake` provider unless `SAGASMITH_ALLOW_FAKE_PROVIDER=1`. |
| T-02-12 | Tampering | Unvalidated parsed_json reaching caller | mitigate | `invoke_with_retry` runs `jsonschema.validate` before returning; failures flow through the retry ladder. |
