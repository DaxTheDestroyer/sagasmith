# Phase 5 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Textual TUI / none for shadcn |
| Preset | not applicable |
| Component library | Textual widgets |
| Icon library | none |
| Font | System terminal monospace |

**Source:** User objective and project specs lock this phase to the existing Python Textual TUI. No React, web UI, shadcn, Tailwind, or external registry is allowed for Phase 5.

**Existing shell to preserve:** `SagaSmithApp` composes `SafetyBar` at top, `NarrationArea` left, `StatusPanel` right, and `InputLine` bottom. Phase 5 extends this shell rather than replacing it.

**Primary focal point:** The primary play-screen visual anchor is the latest deterministic mechanical outcome or active combatant state. Narration remains the dominant reading surface, the right status panel carries persistent mechanics context, and the safety bar remains continuously visible above both.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline separators in dice rows, compact label/value gaps |
| sm | 8px | Compact element spacing inside roll detail rows and sheet stat groups |
| md | 16px | Default modal padding, status panel groups, command result spacing |
| lg | 24px | Character sheet section separation and dice overlay vertical rhythm |
| xl | 32px | Main layout split between narration and status concerns when translating to CSS cells |
| 2xl | 48px | Major overlay content blocks: roll formula, outcome, audit reference |
| 3xl | 64px | Reserved page-level spacing; do not add new Phase 5 page layouts |

Exceptions: Textual CSS uses terminal cells, not physical CSS pixels. Map the declared scale to cell counts as follows: 4px = no extra cell/inline separator, 8px = 1 cell, 16px = 2 cells, 24px = 3 cells, 32px = 4 cells, 48px = 6 cells, 64px = 8 cells. Do not introduce non-scale values.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px equivalent | regular (400) | 1.5 |
| Label | 14px equivalent | semibold (600) | 1.2 |
| Heading | 20px equivalent | semibold (600) | 1.2 |
| Display | 28px equivalent | semibold (600) | 1.2 |

**Textual mapping:** Use terminal monospace text. Use Rich/Textual style emphasis only for the two declared weights: regular and bold/semibold. Do not add italic, underline, all-caps-only warnings, or additional type sizes for Phase 5.

**Source/default:** Sizes and line heights use the GSD default typography contract adapted to terminal rendering. Existing `RichLog(markup=False)` safety remains mandatory for transcript/narration content.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0B0F14` | App background, narration body, modal backdrop, ordinary transcript text surface |
| Secondary (30%) | `#17202A` | Status panel, character sheet panel/modal surface, dice overlay surface, grouped stat blocks |
| Accent (10%) | `#D6A84F` | Dice result total, degree badge, active combatant marker, `/sheet` section headings, roll audit reference label |
| Destructive | `#C75252` | Destructive actions only; Phase 5 has no destructive action implementation |

Accent reserved for: dice result total, degree badge, active combatant marker, `/sheet` section headings, and roll audit reference label. Accent must not be applied to every command, every input, or all interactive elements.

**Additional semantic colors:**
- Success/check-positive: `#6FAF75` for success and critical success labels only.
- Failure/check-negative: `#C75252` for failure, critical failure, blocked targeting, and invalid action-count messages.
- Warning/safety: preserve existing Textual `$warning-darken-1` safety bar styling so `/pause` and `/line` remain visually distinct from dice/combat accents.

**60/30/10 contract:** The narration and app background remain visually dominant. Panels and modals occupy secondary surface treatment. Gold accent is sparse and reserved for auditable mechanics highlights.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Roll check |
| Empty state heading | No rolls yet |
| Empty state body | Your first mechanical check will appear here after you attempt a risky action. Type an action or use `/sheet` to review your character. |
| Error state | This check could not be resolved because required rules data is missing. No roll was recorded; try another action or type `/sheet` to inspect the current character. |
| Destructive confirmation | None in Phase 5. `/retcon` remains an acknowledge-only Phase 8 stub and must not present rollback confirmation in this phase. |

**Reveal-mode dice overlay copy:**
- Title: `Reveal Check`
- Formula row label: `d20 + modifier vs DC`
- Pre-reveal prompt: `Press Enter to reveal. /pause and /line remain available.`
- Result summary: `{degree}: d20 {natural} + {modifier} = {total} vs DC {dc}`
- Audit link/cue: `Saved to roll log: {roll_id}`
- Dismiss hint: `Enter: continue  Esc: close details  /pause: pause  /line: set a line`

Note: Phase 5 checks are pre-resolved by deterministic services before display, so the
prompt uses "reveal" rather than "roll" — the d20 has already been rolled and persisted
when this surface is rendered.

**Character sheet copy for `/sheet`:**
- Title: `Character Sheet`
- Empty/missing sheet: `No pregenerated character is loaded yet. Phase 5 requires one level-1 martial pregen before checks or combat begin.`
- Footer hint: `Esc closes sheet. Rolls and HP changes remain in the transcript and roll log.`

**Combat/action copy:**
- Invalid action count: `No actions remaining for {combatant}. End the turn or choose a reaction if available.`
- Invalid position: `{target} is {position}; this action requires close or near range.`
- HP delta: `{target}: HP {old_hp} -> {new_hp} ({delta_label})`

## Reveal-Mode Dice Interaction Contract

1. **Trigger:** When `dice_ux=reveal` and a skill check, Perception initiative, Strike, or other Phase 5 supported check resolves, render the reveal block before outcome narration continues. Phase 5 satisfies this with a transcript-rendered reveal block (a stable, plain-text block written to `NarrationArea` with title `Reveal Check` and the required fields below) followed by the compact persisted transcript line. A Textual `ModalScreen` is **not required** for Phase 5; modal interaction is deferred to a later phase.
2. **Required fields:** The reveal block must display all of: `DC`, `modifier`, `d20 result`, `total`, `degree`, `roll_id`, and the check reason/stat when available.
3. **Roll timing:** Phase 5 checks are deterministic and pre-resolved by `RulesEngine`/`CombatEngine` before display. The reveal block reveals an already-recorded `RollResult` and must not generate a second roll on display. If a future phase introduces an interactive modal, pressing Enter must reveal the precomputed result rather than invoke a fresh roll.
4. **Keyboard dismissal:** Phase 5 transcript-rendered reveal does not require a modal dismissal interaction; the reveal block is appended to scrollback and the input line retains focus. The `Esc`/`Enter` dismiss hint copy is reserved for the future modal implementation and may be omitted from the Phase 5 transcript output.
5. **Safety controls remain accessible:** `/pause` and `/line` must remain available throughout reveal rendering. Because Phase 5 does not capture input in a modal, no passthrough bindings are needed; safety controls operate through the normal input line.
6. **Non-blocking behavior:** Reveal rendering must not block safety interrupts, budget hard-stop display, or persistence of the roll log.
7. **Scrollback persistence:** Append a compact transcript line: `[roll] {reason}: d20 {natural} + {modifier} = {total} vs DC {dc} — {degree} ({roll_id})`. This line must be persisted with scrollback so the player can audit it later.
8. **Audit path:** The transcript line's `{roll_id}` must match the persisted roll log entry. Do not use a UI-only identifier.
9. **Hidden/auto modes:** `hidden` must not render the reveal block. `auto` may flash a compact result for 500 ms per GAME_SPEC, but Phase 5 acceptance centers on reveal mode.
10. **No LLM math:** The reveal block must only render values from deterministic services/models (`CheckResult`, `RollResult`, combat state), never values authored in narration text.

## Character Sheet `/sheet` Contract

`/sheet` must provide a readable, read-only view of the one Phase 5 pregenerated level-1 martial PC.

Required groups, in order:
1. Identity: name, level, ancestry, background, class.
2. Durability: current HP / max HP, armor class, conditions.
3. Perception and saves: Perception modifier, Fortitude, Reflex, Will.
4. Skills: the four trained skills first, then any other supported skill modifiers needed by the slice.
5. Attacks: one melee Strike and one ranged Strike with attack modifier, damage expression, traits/range if present.
6. Inventory: only items represented in `CharacterSheet.inventory`; no unsupported equipment actions.
7. Footer: `Esc closes sheet. Type an action when ready.`

Interaction rules:
- `/sheet` opens from play and combat states without consuming combat actions.
- `Esc` closes and returns focus to the input line.
- Sheet display must not expose GM-only enemy data or hidden DCs.
- If implemented as transcript output instead of a modal, it must use stable headings and separators so tests can assert field presence.

## Combat and Status Interaction Contract

- Initiative: display initiative order after Perception initiative resolves and persist order through checkpoint/resume. Active combatant uses the accent marker only.
- Actions: show `Actions: 3/3`, `2/3`, `1/3`, or `0/3` for the active combatant. Reaction shows `Reaction: available` or `Reaction: spent`.
- Positions: render only `close`, `near`, `far`, and `behind_cover`. Do not introduce tactical grid, coordinates, or map UI.
- Strikes: show attack roll through the reveal overlay when configured; damage and HP deltas appear in status/transcript after the roll result.
- Enemy count: render no more than two enemies in the Phase 5 combat panel/status text.
- Last rolls: right status panel shows the last three compact roll summaries, newest first or newest last consistently; each includes degree and roll id suffix.

---

## Accessibility and Testability Contract

- All Phase 5 mechanics outputs must be readable as plain text in terminal snapshots; no essential information may be color-only.
- Degree labels must use exact strings: `critical_success`, `success`, `failure`, `critical_failure` in storage/audit contexts; UI may display title-case in addition, not instead.
- Roll and sheet views must be deterministic under tests: stable ordering, stable headings, no time-dependent animation requirement in assertions.
- Rich markup remains disabled for untrusted transcript/narration lines. Mechanics summaries should be built from structured data, not interpolated LLM prose.
- UI tests should be able to assert visible text for DC, modifier, natural d20, total, degree, and roll id for TUI-07.
