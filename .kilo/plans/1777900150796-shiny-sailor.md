## Current Friction

The current `author_session` Interface is Shallow: callers pass `session_number`, `campaign_id`, `db_conn`, `vault_service`, optional `history`, and `date_in_game`, then receive only a path. Behind that small return value, the Implementation performs source selection, raw SQLite transcript and roll reads, page drafting, direct master-vault write, and leaves derived-index/player-vault ordering to `GraphRuntime`.

The deletion test shows the mixed shape: deleting the draft formatting would scatter useful page knowledge, but deleting the direct vault write would concentrate complexity in the existing audited vault-write Module. That means the useful Module is the draft authoring logic, not the file-writing side effect.

`CanonicalTurnHistory` has Depth for canonical turn IDs, but session page source rows still leak raw SQL into Session Page Authoring. This keeps canonical filtering partially local and partially duplicated.

`GraphRuntime._author_session_after_close()` manually calls `resolver.refresh()`, `rebuild_indices()`, and `sync()` after a direct `vault_service.write_page()` call. This bypasses the vault-write audit Seam used by turn close and gives weak Locality for failure semantics.

The existing `scene_brief` beat extraction is also fuzzy: session authoring reads transcript rows with kind `scene_brief`, while `TranscriptEntry.kind` does not currently allow that value. This plan preserves current behavior for loose/raw rows but does not broaden the transcript schema unless a separate product decision promotes scene-brief persistence.

## ADR Fit

This plan does not contradict ADR-0001. LangGraph nodes remain pure Adapters, and persistence remains below orchestration. The change deepens deterministic Modules around Canonical Turn History, Session Page Authoring, and audited vault writes.

## Target Shape

Create a deeper Session Page Authoring Module whose Interface drafts a `VaultPage` from canonical session history and does not write files.

Move session page source-row selection into Canonical Turn History so canonical filtering, source ordering, and retconned-turn exclusion have Locality.

Expose a reusable audited vault-write Module in `turn_close.py` so normal turn-close vault pages and session pages cross the same Seam for write, audit, derived-index update, and player-vault sync.

Keep `GraphRuntime` as the orchestration Adapter: it detects session end, asks the Session Page Authoring Module for a draft, and applies that draft through the audited vault-write Module.

## Implementation Steps

1. Add SagaSmith domain vocabulary.

   Add `Session Page Authoring` to `docs/sagasmith/SAGASMITH_CONTEXT.md` because the deepened Module is named after a SagaSmith concept not yet defined there. Keep architecture-review notes out of this product context file.

2. Deepen Canonical Turn History.

   In `src/sagasmith/persistence/turn_history.py`, add small frozen data values for session page source rows, likely `SessionTranscriptRow`, `SessionRollRow`, and `SessionPageSource`.

   Add `CanonicalTurnHistory.session_page_source(campaign_id, session_id, include_retconned=False) -> SessionPageSource`.

   Implement source reads with JOINs against `turn_records` so the canonical predicate is owned here, not inside Session Page Authoring. Order transcript rows by canonical turn order then transcript sequence. Order roll rows by canonical turn order then roll timestamp and roll id.

   Keep these row values independent from `TranscriptEntry` because session page source may include loose historical row kinds such as `scene_brief` that are not part of the persisted `TranscriptEntry` model Interface.

3. Convert Session Page Authoring to draft-only.

   In `src/sagasmith/agents/archivist/skills/session_page_authoring/logic.py`, replace the current write-oriented `author_session(...) -> str` with a draft-oriented Interface such as `draft_session_page(session_number, campaign_id, history, date_in_game="unknown") -> VaultPage`.

   The Implementation should call `CanonicalTurnHistory.session_page_source(...)`, extract wikilinks, build `SessionFrontmatter`, and build the markdown body. It should not accept `VaultService`, should not accept a raw SQLite connection, and should not write a file.

   Preserve current page shape: `sessions/session_NNN.md`, `type=session`, `visibility=player_known`, summary, beats, and roll table. Handle `dc is None` safely instead of coercing `None` through `int()`.

   Update imports and tests to use the draft Interface. Do not retain a compatibility wrapper unless another real caller remains.

4. Extract an audited vault-write Module from turn close.

   In `src/sagasmith/persistence/turn_close.py`, extract the existing post-commit vault write/audit/index/sync block into a reusable public Module, likely `apply_vault_writes(conn, turn_record, pages, vault_service) -> None`.

   `close_turn()` should call this Module for `bundle.vault_pages` after the SQLite transaction commits. The new Module should preserve current behavior: write each page atomically to the master vault, append `VaultWriteAuditRecord` only after successful file write, refresh resolver, commit audit rows, update derived indices, sync player vault, record `sync_warning` on sync failure, and mark the turn `needs_vault_repair` on master-vault write failure.

   Keep `_relative_path_for_page()` inside `turn_close.py`; session pages already resolve through page type `session` to `sessions/session_NNN.md`.

5. Route session page writes through the audited Seam.

   In `src/sagasmith/graph/runtime.py`, change `resume_and_close()` so the session-end path passes the completed `TurnRecord` into `_author_session_after_close()`.

   Update `_author_session_after_close()` to create `CanonicalTurnHistory(self.db_conn)`, draft the session page, and call `apply_vault_writes(self.db_conn, completed_record, [session_page], vault_service)`.

   Remove the manual `resolver.refresh()`, `rebuild_indices()`, and `sync()` calls from runtime. Runtime should no longer know vault write ordering details.

6. Update harness backlog after implementation.

   Mark item 6 in `.kilo/ARCHITECTURE-DEEPENING-BACKLOG.md` as implemented, with the concrete files and tests. This is coding-harness output, not SagaSmith runtime context.

## Risks And Controls

Risk: session page authoring currently depends on loose `scene_brief` rows that normal `TranscriptEntry` validation does not allow. Control: keep source rows as raw Canonical Turn History data for this plan, and do not silently expand persisted transcript kinds.

Risk: extracting the audited vault-write Module could alter existing turn-close failure semantics. Control: preserve existing tests first, then add direct tests for the new Interface.

Risk: session-end flow may perform two post-commit vault-write passes when a turn already has normal vault pages. Control: accept this as the minimal correct change for Locality now; future optimization can batch drafts into one write pass if profiling shows it matters.

Risk: session page write failure after the completed turn commit will now surface consistently as `needs_vault_repair`. Control: integration tests should assert audit/write behavior and existing repair guidance remains valid.

## Completion Criteria

- Session Page Authoring has a draft-only Interface with no raw SQLite connection and no vault Adapter in its caller-facing Interface.
- Canonical Turn History owns session page source selection and canonical filtering.
- Session page file writes go through the same audited vault-write Seam as other turn-close pages.
- `GraphRuntime` no longer owns vault write ordering for session pages.
- Tests cover source-row selection, page drafting, audited writing, and integration session-end behavior.
- Backlog item 6 is marked implemented after code and tests pass.
