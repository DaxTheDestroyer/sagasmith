# Plan for Phase 6: AI GM Story Loop

## Objective
Implement the AI GM story loop with three core agents: Oracle (scene planning), RulesLawyer (mechanics conversion), and Orator (narration generation). This phase will enable users to play AI-planned and AI-narrated turns while maintaining strict boundaries between deterministic mechanics and AI-generated content.

## Key Requirements
- AI-01 through AI-10: Core AI agent behaviors
- GRAPH-06/07: Narration recovery mechanisms
- SAFE-01 through SAFE-03: Content safety
- QA-05: Safety regression testing

## Implementation Approach

### 1. Oracle Agent (Scene Planning)
- Implement scene planning that produces validated SceneBrief objects
- Ensure Oracle never emits direct player-facing narration
- Create starting hook generation aligned with onboarding preferences
- Implement replanning capabilities when players accept/reject/bypass beats
- Add small-scope NPC draft creation while preserving future consistency

### 2. RulesLawyer Agent (Mechanics Conversion)
- Convert player intent and scene context into mechanical proposals
- Ensure all mechanical outcomes are deterministic and auditable
- Prevent LLMs from inventing modifiers, DCs, damage, HP changes, action counts, or degrees of success
- Integrate with existing deterministic dice and rules services

### 3. Orator Agent (Narration Generation)
- Render scene plans, memory, player input, and resolved mechanics into second-person prose
- Ensure streaming of at least one complete beat of narration per turn
- Implement dice UX mode respect (auto, reveal, hidden)
- Prevent contradictions between mechanical outcomes and generated narration

### 4. Safety Implementation
- Implement pre-generation content filtering (SAFE-01/02)
- Add post-generation content scanning (SAFE-03)
- Create fallback narration mechanisms for safety violations

### 5. Narration Recovery
- Implement GRAPH-06: Recovery from incomplete narration turns
- Ensure GRAPH-07: Stability of deterministic outcomes during narration retries

## Technical Architecture

### Agent Orchestration
- Utilize existing LangGraph infrastructure
- Implement proper checkpointing before narration streaming
- Ensure state transitions between agents are well-defined

### Memory Integration
- Integrate Archivist agent for bounded memory packets
- Ensure Oracle and Orator receive appropriate context

### Deterministic Boundaries
- Maintain strict separation between AI narrative and rules engine
- Ensure all mechanical computations happen in deterministic services

## Testing and Quality Assurance

### Safety Testing
- Implement QA-05: Regression testing for hard-limit content in player-facing prose
- Verify proper handling of content policy violations

### Integration Testing
- End-to-end testing of the complete AI story loop
- Verification of deterministic outcome stability during narration retries

### Performance Testing
- Ensure narration streaming performance meets latency requirements
- Verify cost governance integration works with AI agents

## Deliverables
1. Implemented Oracle, RulesLawyer, and Orator agents
2. Integrated safety mechanisms (pre/post-generation filtering)
3. Narration recovery mechanisms
4. Updated LangGraph state transitions
5. Comprehensive test coverage including safety regressions
6. Documentation of agent behaviors and boundaries

## Dependencies
- Phase 5: Rules-First PF2e Vertical Slice (completed)
- Existing deterministic services for rules and dice
- LangGraph orchestration framework
- SafetyGuard and CostGovernor services

## Success Criteria
- User can receive 3-5 starting hooks or a curated first-slice hook aligned with onboarding preferences
- Oracle produces validated scene plans and replans around player choices without emitting player-facing narration
- RulesLawyer converts player intent into deterministic mechanical proposals/results without letting LLMs invent game mechanics
- Orator is the only player-facing narrative voice, streams at least one complete beat per completed turn, respects dice UX, and does not contradict resolved mechanics
- Unsafe scene intents or generated prose are blocked, rerouted, retried, or safely degraded, and incomplete narration can be retried or discarded without changing deterministic outcomes
