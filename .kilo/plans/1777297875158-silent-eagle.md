## Architectural Locks (Consumed by Later Phases)

- `CampaignPaths` + `open_campaign()` — single resolve/validate entry point
- `CampaignManifest` (TOML on disk) — human-inspectable metadata, never holds secrets
- `ProviderSettings` + `SettingsRepository` — `SecretRef`-only, `RedactionCanary`-enforced
- `OnboardingWizard` pure-domain state machine — TUI-agnostic; Plan 04 graph runtime drives it via Textual screens
- `OnboardingStore.reload()` returning `OnboardingTriple | None` — supports ONBD-05 re-run without data loss
- `CommandRegistry` + `TUICommand` protocol — stable extension point for Phases 4–8
- `SagaSmithApp` with runtime-bound `.onboarding_store`, `.safety_events`, `.cost_governor` — testable in isolation via direct construction
- `safety_events` table with schema-locked `visibility='player_visible'` in Phase 3 — structural SAFE-06 guarantee
