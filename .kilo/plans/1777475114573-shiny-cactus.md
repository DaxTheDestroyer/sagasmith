### 2. Keep 08-04, But Reclassify It As Final MVP Gate

`07-06-SUMMARY.md` says `QA-08` and `QA-09` were completed, but current code still lacks `run_mvp_smoke`, `SmokeMode.MVP`, `make release-gate`, and `secret-scan` targets. Keep `08-04`, but make it clear that Phase 7 only proved the memory/vault/resume slice while Phase 8 owns the final MVP-wide gate.

Update `08-04-PLAN.md` so it depends on `08-03`, or add a final verification step after `08-03` that re-runs `make release-gate`. The release gate should validate retcon code too, not run before retcon implementation.

### 3. Add Type-Hardening To Release Gate Scope

Phase 7 ended with full `uv run pyright` still failing due to older strict typing errors. Since `make release-gate` must run `make typecheck`, Phase 8 needs one of these explicit plan changes:

- Add a type-hardening task before enabling `release-gate`.
- Or make `08-04` responsible for resolving pre-existing Pyright failures enough for `make typecheck` to pass.

Without this, `QA-09` is planned as release-blocking but likely infeasible.

### 4. Fix `/recap` Canonical Exclusion Gap

`08-01-PLAN.md` Task 2 only updates `get_recent_transcript_context()`, but Phase 7 `/recap` uses `TranscriptRepository.list_recent(limit=5)` directly from `src/sagasmith/tui/commands/control.py`.

Update 08-01 to include:

- `TranscriptRepository.list_recent(..., include_retconned: bool = False)` or a new canonical recent helper.
- `RecapCommand` routed through the canonical helper.
- Tests proving `/recap` does not render retconned transcript rows.

### 5. Strengthen Vault-Write Audit For Actual Rollback

The current `vault_write_audit` plan records only `turn_id`, `vault_path`, operation, and timestamp. That is not enough to restore master-vault files after a retcon because Phase 7 master-vault files are mutable current snapshots and `VaultService.sync()` only projects the current master vault to the player vault.

Update 08-01 Task 3 to record enough data to undo affected vault writes, for example:

- `previous_page_json` or `previous_file_text` before each write, nullable if the file did not exist.
- `new_page_json` or `new_file_text` after each write.
- `content_hash_before` and `content_hash_after` for safety checks.
- A flag or operation value for create/update/delete semantics.

Retcon should block with repair guidance for turns lacking safe vault restoration data rather than pretending `sync()` removes retconned master-vault canon.

### 6. Include Session Page Writes In Audit And Repair

Phase 7 session pages are authored after `close_turn()` in `_author_session_after_close()` and `session_page_authoring.logic.author_session()`, not by `close_turn()` vault pages. Phase 8 must audit and rebuild these too.

Update 08-01 and 08-02 to include:

- Auditing `sessions/session_NNN.md` writes.
- Regenerating affected session pages from canonical completed turns after retcon.
- Tests proving session pages exclude retconned narration, beats, and rolls.

### 7. Use Phase 7 Rebuild Helper

`08-02-PLAN.md` currently plans direct calls to `FTS5Index.rebuild_all`, `reset_vault_graph_cache`, `warm_vault_graph`, and `vault_service.sync()`. Phase 7 added `VaultService.rebuild_indices(conn)`, which refreshes resolver, rebuilds FTS5 when a SQLite connection exists, and warms NetworkX.

Update 08-02 to prefer:

- Restore or rebuild canonical master-vault state.
- `vault_service.rebuild_indices(db_conn)`.
- `vault_service.sync()`.

### 8. Add Rolling Summary And Memory Canon Tests

Retcon checkpoint rewind should restore graph `rolling_summary`, but Phase 7 also writes a master-vault meta rolling summary page. Update 08-02 tests to prove:

- In-memory graph `rolling_summary` after retcon is from the prior checkpoint.
- `meta/rolling_summary.md`, memory packet assembly, and `/recap` do not include retconned facts.

### 9. Clarify Retcon Runtime API For TUI

`08-03-PLAN.md` references `preview_retcon/list_candidates`, but its interface snippet only defines `preview_retcon` and `confirm_retcon`.

Update 08-02 and 08-03 to define one clear API, preferably:

- `graph_runtime.list_retcon_candidates(limit: int = 5)`
- `graph_runtime.preview_retcon(selected_turn_id: str)`
- `graph_runtime.confirm_retcon(selected_turn_id: str, confirmation_token: str, reason: str = "player_retcon")`

Also update the stale command description from “Retcon the last completed turn” to “Choose a completed turn to retcon.”

### 10. Reconcile Audit Failure Semantics

`08-01-PLAN.md` Task 3 says vault-write audit rows are recorded after successful file writes. Add explicit behavior for file-write success followed by audit-write failure.

Recommended handling:

- Treat missing audit metadata as a repair-risk state.
- Mark the turn `needs_vault_repair` or persist `sync_warning` with clear guidance.
- Retcon preview must block if affected vault writes lack restoration metadata.
